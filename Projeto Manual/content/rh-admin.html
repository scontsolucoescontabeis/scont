Administração do módulo RH: cadastro de empresas, empregados, sócios e rubricas, além das configurações que alimentam o Controle de Frequência, os Lançamentos e o Renomeador de Arquivos. É a base de dados de onde todas as outras ferramentas do módulo RH puxam informação.

Empresas

Como usar

  1. Acesse a aba Empresas no menu lateral.
  2. Use os filtros de Código/Nome e Status para localizar uma empresa na tabela.
  3. Para atualizar o cadastro em massa, vá em Importar Dados → Empresas, arraste o relatório exportado do ERP e escolha entre Substituir (apaga tudo e recria) ou Acrescentar (atualiza/insere sem apagar o resto).
  4. Se alguma empresa ficar sem UF preenchida, use o botão 🌍 Preencher UF pendentes na aba de importação de Empresas.

Funcionalidades

  • Listagem paginada com busca por código/nome e filtro por situação (Ativo/Inativo).
  • Importação em lote a partir do relatório nativo "Relação de Empresas" exportado direto do ERP — não é mais um modelo de planilha manual.
  • Preenchimento automático da UF a partir do CEP de cada empresa (sem precisar digitar).
  • Modo de importação com dois caminhos: substituir a base inteira ou apenas atualizar/inserir sem apagar nada.

Memórias de cálculo

A importação lê o relatório exportado do ERP a partir da linha 8 (7 linhas de título/paginação são ignoradas). Os dados vêm intercalados com colunas em branco (resquício de células mescladas do relatório original), então a leitura é posicional, coluna a coluna:

nome_empresa ← vem da coluna "Razão Social", não da coluna "Empresa" (a coluna "Empresa" costuma vir truncada em parte das linhas) email ← novo campo, mapeado da coluna "Email" do relatório Data de Encerramento e códigos internos do ERP → ignorados, não importados

Preenchimento de UF: para cada linha com CEP preenchido, o sistema consulta a API pública ViaCEP; se a API falhar ou não responder, cai para uma tabela local de faixas numéricas de CEP por estado (ex.: CEPs entre 01000-000 e 19999-999 → SP). Se nada resolver, a UF fica em branco.

No modo Acrescentar, se a busca de UF falhar para uma empresa que já tinha UF salva, o valor antigo é preservado (nunca é sobrescrito por vazio). No modo Substituir, a UF sempre reflete o resultado da importação atual — inclusive ficando vazia se a busca falhar.

Perguntas frequentes

Por que uma empresa importada ficou com o nome diferente do que aparece no ERP?
O nome usado é sempre o da coluna "Razão Social" do relatório, que é mais confiável do que a coluna "Empresa" (que vem truncada em cerca de 28% das linhas do relatório).
Preciso preencher a UF manualmente?
Não — ela é sempre derivada do CEP, automaticamente, na importação ou pelo botão "Preencher UF pendentes". Não existe campo de edição manual de UF.

Empregados

Como usar

  1. Acesse a aba Empregados.
  2. Filtre por empresa (uma ou várias, com busca), por situação (Trabalhando, Demitido, afastamentos etc.) e por tipo de empregado (Empregado, Contribuinte, Estagiário).
  3. Abra "📋 Selecionar Campos do Relatório" para escolher quais colunas exportar/imprimir, depois use 📥 Exportar Excel ou 🖨️ Gerar PDF.
  4. Para atualizar em massa, use Importar Dados → Empregados com o relatório de empregados exportado do ERP.
  5. Para adicionar um único empregado manualmente, abra "➕ Adicionar Novo Empregado" e informe empresa, código e nome.

Funcionalidades

  • Tabela paginada com filtro combinado por empresa(s), situação e tipo de empregado.
  • Seleção de campos do relatório (mais de 60 campos possíveis, incluindo endereço, documentos, dependentes e dados bancários) para exportação em Excel ou PDF.
  • Importação em lote com detecção automática de colunas pelo nome do cabeçalho — tolerante a arquivos com uma empresa por bloco.
  • Cadastro manual rápido (empresa + código + nome) para casos avulsos.

Memórias de cálculo

A importação de empregados lê a planilha por nome de coluna, não por posição fixa, porque o relatório do ERP repete a linha de cabeçalho a cada troca de empresa dentro do mesmo arquivo (uma exportação com várias empresas gera vários blocos, cada um com seu próprio cabeçalho). O sistema:

1. Percorre o arquivo linha a linha. 2. Ao encontrar uma linha cujo primeiro campo é exatamente "Cód Emp", reconhece como cabeçalho e remonta o mapa "posição da coluna → campo do sistema" com base nos rótulos daquela linha (não gera empregado para essa linha). 3. Linhas seguintes usam esse mapa até o próximo cabeçalho aparecer. 4. Rótulos repetidos no arquivo (ex.: "Categoria" aparece tanto para a categoria do empregado quanto para a categoria da CNH) são resolvidos pela ordem de aparição: a 1ª ocorrência mapeia para a 1ª do modelo, a 2ª para a 2ª.

Isso evita dois problemas do relatório real: linhas de cabeçalho repetidas virarem "empregados fantasmas", e a ordem das colunas de dados bancários (Agência/Conta) mudar de empresa para empresa dentro do mesmo arquivo.

Perguntas frequentes

A importação apaga os empregados que não estão na planilha nova?
Depende do modo escolhido no diálogo de importação: "Substituir" apaga e recria; "Acrescentar" só atualiza/insere, sem excluir ninguém.

Informações de Férias

Como usar

  1. Importe o PDF "Relação de Férias Calculadas" (todas as empresas de uma vez) em Importar Dados → Férias.
  2. Consulte os períodos aquisitivo e de gozo de cada empregado na aba Informações de Férias, com busca por empresa ou empregado.

Funcionalidades

  • Extração automática, a partir do PDF, de: empresa, empregado, início/fim do período aquisitivo e início/fim das férias.
  • Esses dados alimentam o badge "Férias" exibido no Controle de Frequência para o período em que o empregado estiver de férias.

Perguntas frequentes

Dá para editar manualmente um período de férias importado?
Não — assim como a Jornada, os dados de férias só entram via importação do PDF; uma nova importação substitui o conteúdo anterior.

Jornada de Trabalho

Como usar

  1. Importe o PDF "Horário de Trabalho" (todas as empresas) em Importar Dados → Jornada de Trabalho.
  2. Consulte na aba Jornada de Trabalho os horários de cada empregado, um por linha, com os 7 dias da semana em colunas.
  3. Para corrigir um horário pontual sem reimportar tudo, clique em ✏️ Editar na linha do empregado.

    4. Funcionalidades

    • Importação em massa via PDF: um empregado por linha, com entrada/intervalo/saída de cada dia em que trabalha.
    • Edição manual pontual: marcar/desmarcar um dia como "não trabalha" (folga) e ajustar entrada, intervalo (início/fim) e saída dos dias trabalhados, sem precisar reimportar o PDF inteiro.
    • Essa jornada é a base usada pelo Controle de Frequência para calcular horas trabalhadas, atrasos e horas extras.

    Memórias de cálculo

    A importação do PDF é um snapshot completo: cada nova importação apaga e recria toda a tabela de jornada. Uma edição manual feita antes é perdida se aquele mesmo empregado aparecer de novo no próximo PDF importado — é um efeito colateral aceito, não um bug.

    Na edição manual, a regra de validação é: se o dia está marcado como "Trabalha", Entrada e Saída são obrigatórios; o intervalo (início/fim) é opcional, mas não pode ser preenchido só de um lado.

    Perguntas frequentes

    Editei a jornada manualmente e depois importei um novo PDF — o que aconteceu com a edição?
    Foi substituída. A importação de PDF sempre recria a tabela inteira; edições manuais só sobrevivem até a próxima importação.
    A edição manual cadastra um empregado novo?
    Não — só edita horários de quem já está na lista (trazido por uma importação anterior). Empregado novo entra sempre via PDF.

Sócios

Como usar

  1. Consulte a aba Sócios, com filtro por nome, CPF, empresa e situação da empresa (ativa/inativa).
  2. Para atualizar em massa, use Importar Dados → Sócios (Excel/CSV) — baixe o modelo pelo botão "⬇️ Modelo" antes de preencher.

Funcionalidades

  • Cadastro de quadro societário: nome, CPF, participação, cargo, datas de ingresso/saída e e-mail de cada sócio, vinculados à empresa.
  • Importação também atualiza dados da própria empresa relacionados ao quadro societário (capital social, e-mail, data de atualização do quadro).

Rubricas

Como usar

  1. Filtre por empresa e/ou texto (código ou descrição) na aba Rubricas.
  2. Para cadastrar uma rubrica avulsa, use "➕ Adicionar / Editar Rubrica" (empresa, código, descrição e tipo).
  3. Para atualizar em massa, use 📥 Importar Excel (modelo disponível em "⬇️ Modelo").

Funcionalidades

  • Catálogo de rubricas por empresa: código, descrição e tipo (ex.: Vencimento, Desconto).
  • É esse catálogo que o Controle de Frequência e os Lançamentos usam para detectar automaticamente rubricas de falta e de DSR por falta, a partir do texto da descrição — sem precisar configurar um código fixo por empresa.

Memórias de cálculo

Detecção automática de rubricas de falta (usada no Controle de Frequência e nos Lançamentos): a descrição de cada rubrica do catálogo é normalizada (maiúsculas, sem ponto final) e comparada por conteúdo:

contém "DIAS FALTAS" e NÃO contém "DSR" → rubrica de falta normal contém "DIAS FALTAS" e contém "DSR" → rubrica de desconto de DSR por falta

Cobre variações já vistas no catálogo real, como DIAS FALTAS, DIAS FALTAS 1, DIAS FALTAS DSR, DIAS FALTAS DSR.. Se uma empresa não tiver a rubrica de DSR cadastrada aqui, o cálculo automático de desconto de DSR é pulado só para ela (as demais empresas do lote continuam normalmente), com aviso ao final da geração do TXT.

Perguntas frequentes

Esse cadastro de rubricas é o mesmo dos "códigos de rubrica" configurados na tela de exportação de TXT do Controle de Frequência?
Não. Este é o catálogo geral de rubricas da empresa (o que existe no ERP). A configuração de qual código usar para cada evento (Horas Extras, Falta etc.) ao gerar o TXT é feita em outra tela, dentro do próprio Controle de Frequência.

Regras de Renomeação de Arquivos

Como usar

  1. Em "1. Padrões de Renomeação", cadastre um padrão DE (como o arquivo chega) e um padrão PARA (como ele deve ficar), usando as tags disponíveis.
  2. Em "2. Dicionário de Nomes de Documentos", associe o texto cru extraído do nome original a um nome final mais legível (ex.: ExtratoMensal → Folha de Pagamento).
  3. Os padrões e o dicionário cadastrados aqui são consumidos pela ferramenta Conversor e Renomeador de Arquivos — é lá que a renomeação de fato acontece.

Funcionalidades

  • Tags disponíveis: {CODIGO_EMPRESA}, {NOME_EMPRESA}, {NOME_ARQUIVO}, {NOME_DOCUMENTO}, {MM}, {AAAA}, {IGNORAR}.
  • Vários padrões podem ficar cadastrados ao mesmo tempo, para diferentes formatos de arquivo recebidos.

Memórias de cálculo

O padrão DE é um molde para extrair pedaços do nome original: {CODIGO_EMPRESA} e {NOME_ARQUIVO} capturam texto; {IGNORAR} descarta um trecho sem usá-lo. O padrão PARA monta o nome novo substituindo cada tag pelo valor capturado (ou, no caso de {NOME_EMPRESA} e {NOME_DOCUMENTO}, por valores buscados no cadastro de empresas / dicionário de nomes):

DE: {CODIGO_EMPRESA}_{NOME_ARQUIVO}_{IGNORAR}_{MM}{AAAA} PARA: {NOME_DOCUMENTO}_{NOME_EMPRESA}_{AAAA}_{MM} Exemplo: "012_ExtratoMensal_v2_072026.pdf" código da empresa = 012 nome do arquivo = ExtratoMensal → dicionário → "Folha de Pagamento" trecho "v2" = ignorado mês/ano = 07/2026 resultado: "Folha de Pagamento_Nome da Empresa Ltda_2026_07.pdf"

Perguntas frequentes

Preciso cadastrar o mapeamento no dicionário mesmo se o padrão PARA não usar {NOME_DOCUMENTO}?
Não — o dicionário só é consultado quando a tag {NOME_DOCUMENTO} aparece no padrão PARA.

Importar Dados

Como usar

  1. Acesse a aba Importar Dados: cada cartão (Empresas, Empregados, Férias, Jornada, Rubricas, Sócios) importa um tipo de cadastro separadamente.
  2. Baixe o modelo quando disponível (botão "⬇️ Modelo"), preencha e arraste o arquivo — ou clique para selecionar.
  3. Quando o sistema perguntar o modo de importação, escolha Substituir (apaga os dados anteriores daquele tipo e importa os novos) ou Acrescentar (mantém o que já existe e atualiza/insere).

Funcionalidades

  • Empresas e Empregados: leem o relatório exportado diretamente do ERP (não é mais um modelo simples de planilha).
  • Rubricas e Sócios: leem planilha no modelo próprio do sistema, baixável na tela.
  • Férias e Jornada: leem PDF exportado do sistema fonte (não planilha).
  • Barra de progresso e registro da data/hora da última importação de cada tipo.

Memórias de cálculo

Regra geral: nenhuma importação exclui um registro que não esteja na planilha/PDF novo, a menos que o modo "Substituir" seja escolhido explicitamente. O modo "Acrescentar" (upsert) sempre atualiza pela chave de identificação de cada cadastro (ex.: código da empresa, ou empresa + empregado), sem apagar o que não veio no arquivo.
