Comportamento dos telefones

  • Novo telefone — abre um modal pedindo nome e telefone. Pode gerar aleatório (ex: +55 11 9XXXX-XXXX) ou inserir manualmente. Salva no localStorage.
  • Primeira mensagem — ao enviar, a edge function dev-message cria o contato e a conversa no banco. O simulador salva o conversa_id recebido para aquele telefone.
  • Realtime por cliente — cada cliente tem sua própria subscription Supabase Realtime filtrada pelo conversa_id. Mensagens de BOT e AGENTE aparecem automaticamente.
  • Badge de não-lidas — mensagens de BOT/AGENTE que chegam enquanto o cliente não está selecionado mostram o contador verde na sidebar.
  • Status da conversa — chip no header mostra ABERTA / EM_ATENDIMENTO / ENCERRADA em tempo real.
  • Excluir cliente — remove do localStorage. A conversa fica no banco (não apaga dados reais).
