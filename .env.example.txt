Aja como um arquiteto de software e desenvolvedor principal para a empresa SCONT - Soluções Contábeis. Estamos estruturando um portal de aplicações que servirá como a página de redirecionamento central para todas as outras aplicações da empresa.

Requisitos Fundamentais:

Acesso ao Portal:O acesso ao portal será unicamente via login.
Um administrador do portal gerenciará o acesso.


Gerenciamento de Acesso pelo Administrador:O administrador receberá solicitações de acesso de usuários.
O administrador avaliará, aprovará ou rejeitará cada solicitação.
Em caso de aprovação, o administrador especificará quais aplicações o usuário poderá acessar no portal.


Experiência do Usuário:Após o login, o usuário deverá visualizar apenas as aplicações para as quais tem permissão de acesso. Aplicações sem permissão não devem ser exibidas.


Arquivos de Tela (fornecidos no contexto):admin-login.txt: Código da tela de login do administrador.
admin-dashboard.txt: Código da tela de administração (dashboard).
login.txt: Código da tela de login do usuário.
portal.txt: Código da tela principal do portal para o usuário logado.


Tecnologias a serem utilizadas:Supabase: Para autenticação, banco de dados e possivelmente funções serverless/edge.
Resend: Para envio de notificações por e-mail (ex: confirmação de solicitação, aprovação/rejeição).

Tarefa:

Dê continuidade à elaboração e arquitetura desta plataforma. Detalhe os próximos passos técnicos para a construção, focando na integração das funcionalidades descritas com as tecnologias Supabase e Resend, e considerando os arquivos de tela fornecidos.

Entregáveis Esperados:

Um plano de arquitetura detalhado para o portal e seu sistema de gerenciamento de acesso.
Como Supabase será configurado e utilizado para gerenciar usuários, autenticação, permissões (roles) e o banco de dados das aplicações.
Como Resend será integrado para automação de e-mails relacionados ao fluxo de solicitação e aprovação de acesso.
Uma proposta de estrutura de API (endpoints) que a aplicação backend precisará expor para o frontend interagir, especialmente para gerenciamento de usuários e permissões.
Sugestões de próximos passos de desenvolvimento (ex: configuração inicial do Supabase, desenvolvimento do fluxo de login, criação do painel de admin).

Usuário acessa login.html → cria conta e solicita acesso ao portal → Admin em admin-dashboard.html aprova → update status + insert user_permissions + e-mail via Resend. → Usuário loga → vê  e acessa sem necessidade de nova autenticação os apps liberados pelo admin.
